## Fluxo de uso ponta-a-ponta

1. HR loga → `/painel` mostra banners de **setup** (se incompleto), **planos pendentes**, **outcomes para atribuir**, **falha na geração** (se houver)
2. Header tem badge `⚙ N sugestões` quando fact-extractor capturou fatos novos
3. `/configuracoes/perfil` → "🪄 Sugerir valores" → IA pré-preenche → HR ajusta e salva
4. HR cria pesquisa em `/gerenciar-pesquisas`, importa CSV, dispara magic links
5. Pesquisa fecha → orchestrator roda automaticamente:
   - `computeOutcomes()` fecha outcomes pendentes (com delta normalizado por scoring_direction)
   - Analyst → Curator (com hard-filters TS) → Consultant gera planos shape v2
   - Insere em `action_plans` + cria `action_outcomes` para cada plano
6. HR vai em `/planos-acao`, clica num plano → vê `PlanV2` completo (roadmap, vendors, RACI, KPIs, riscos)
7. Botão flutuante "💬 Perguntar sobre este plano" abre drawer com chat contextual (streaming)
8. HR clica "Aprovar" → task criada no Kanban com `action_plan_id` linkado
9. HR move task no Kanban → quando solta em "Concluído", modal pede outcome qualitativo → cria `company_actions_taken` (source='sentinel_kanban')
10. Quando próxima pesqusa fechar → outcomes calculados → modal sequencial de atribuição → o que funcionou vira `preferred` no próximo Curator
11. `/relatorios/eficacia` mostra tabela com export PDF/CSV
12. `/assistente` para conversar com IA da empresa (visão geral, simulações, pitch executivo) — streaming

## Convenções importantes

- **PT-BR em todo lugar** (UI, prompts, mensagens)
- **RLS scoped por `company_id`** — funções `get_user_role()` e `get_user_company_id()` (migration 002)
- **Server endpoints** validam role HR/ADMIN antes de mutar
- **Admin client** (`createAdminClient`) só no orchestrator/pipeline (bypass RLS)
- **shadcn/ui**: `Button`, `Card`, `Input`, `Label`, `Checkbox`, `Badge`, `Dialog`, `Dropdown`, `Sheet`. Sem `Textarea`/`Tabs`/`Select` shadcn — uso `<textarea>`, botões e `<select>` nativos.
- **Delta normalizado em `action_outcomes`**: positivo = melhora, negativo = piora — independente de `scoring_direction` da dimensão. Calculado em `learning/outcomes.ts`.
- **Streaming chat**: cliente usa helper `lib/chat-stream.ts` que abstrai SSE. Body opcional `stream: true`; fallback síncrono funciona.

## Testes recomendados antes de produção

Como o app tem zero casos reais, vale validar manualmente:

1. **Migration 008** aplica limpa em Supabase (CREATE TYPE ADD VALUE pode pedir COMMIT separado em algumas versões)
2. **Pipeline completo**: criar empresa → preencher perfil 80% → criar pesquisa → simular CSV de respostas → fechar pesquisa → verificar planos shape v2 + outcomes pendentes
3. **Approve flow**: aprovar plano → checar task no Kanban → mover para Concluído → modal de outcome → checar `company_actions_taken`
4. **Reject flow**: rejeitar plano → motivo → checar `profile_events` + `company_profiles.constraints` atualizado
5. **Próximo ciclo**: criar 2ª pesquisa → fechar → `computeOutcomes()` deve calcular delta da 1ª → banner de atribuição aparece
6. **Chat streaming**: abrir `/assistente`, mandar mensagem, ver texto aparecer em chunks
7. **Chat por plano**: abrir um plano, clicar "💬", perguntar "alternativa sem app?"
8. **Sugestões IA**: dizer algo factual no chat ("nosso budget é R$50k"), fechar chat, abrir badge `⚙ 1 sugestões`, aceitar, verificar perfil atualizado
9. **Export**: `/relatorios/eficacia` → baixar CSV + PDF
